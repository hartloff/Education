== Minimum Viable Product (MVP)
Practical Objective: Deliver a bug-free and great MVP
Verify: The course staff will follow your readme to run your software and use it. If there are any issues or bugs at all, this objective has not been met. Your software should seem as if it's ready for release and use for its intended purpose. It must be easy and intuitive to use without any supporting documentation.
Technical Requirement: Develop functional software that is ready for use.
Verify: The course staff will verify that your software delivers the stated MVP functionality from the team contract.
Technical Requirement: Create a release of your MVP on GitHub as "v0.2.0"
Verify: This release will appear on your project page

Nothing has to be polished in the MVP, but it MUST work. A user must be able to use your software for the purpose stated in the design document. No add-on features are required. The interface doesn't have to be clean, but again it needs to be functional.

To finalize your MVP it must be merged into your master branch and be tagged as release "v0.2.0" on GitHub.

== Readme
Requirement: Write a readme for your project.
Verify: The course staff will follow the exactly and check if your software runs.

Add a readme to your project describing how to install your software from source. You can assume these readers have coding experience. You should include any relevant instructions such as installing dependencies, how compile/run the code, and any platform on which it will or will not function. Do not skip any details needed for a programmer to get from cloning your repo to running your software. Once it's running you can assume the user can follow your video to learn to how to use the software so usage instructions are not necessary.

== YouTube Video
Requirement: Upload a video demo of your MVP to YouTube.
Verify: This video will appear on your project page. Course staff will verify that it meets the requirements

As with the prototype, you must also complete a video demo of your MVP and upload it to YouTube. This video must show a working demo of all features in your MVP as described in your team contract. This video will be directed towards a non-technical audience. You should not show any code or use any technical language in this video.

This video must be shorter than 2 minutes. The goal is to demo your functioning software to the end users.


Submission

Once complete, submit the link to your MVP video here. After submitting, check your project page to verify that the video is displayed properly.
