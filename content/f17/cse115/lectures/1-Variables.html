== Supplements Variables Concept Slides == Variables Introduction:

Computer programs manipulate data. Data is stored in a computer's memory. A variable is a location in the computer's memory that has a name.

In Java variables must be declared before they can be used. A declaration gives a variable both a and a . The type of a variable says what kind of value will be stored in the variable.

For example, 

    int myFirstVariable;
declares a variable with name myFirstVariable and type int. int denotes whole numbers, like 17, -3, and 0 (and many more!)

The order is important: the type comes first, followed by the name of the variable. The semicolon ';' at the end of the line marks the end if the declaration.

An assignment is used to store a value in a variable. For example, to store the value 7 in the variable myFirstVariable we write: 

    myFirstVariable = 7;

The order is again important: we first write the name of the variable, then the assignment operator ('='), followed by the value to be stored in the variable. The semicolon marks the end of the assignment. In the example above the declaration of the variable and the assignment of a value to the variable are done in two separate instructions.

We can combine these two steps into one, which is sometimes a convenience: 

    int myFirstVariable = 7;

Both examples have the same result and follow these two steps:

  1. int myFirstVariable - Declare a variable named myFirstVariable of type int
  2. myFirstVariable = 7 - Assign the value 7 to the variable named myFirstVariable

General structure: 

    [type] [name];
    [name] = [value];


    [type] [name] = [value];

  • A program can have many variables
  • Variables can be reassigned different values
  • Names can contain UPPERCASE and lowercase letter, digits and the underscore '_' character
  • Names cannot contain spaces
  • Names of variables start with a lowercase letter
  • Names of variables follow

Variable Types

Every variable in java must have a type. A variable can only stores values matching its type.

int
Stores integer values (whole numbers). Can store values that are negative, positive, or 0.
double
Stores numbers with decimals (ex. 3.1415). Can store values that are negative, positive, or 0.0. Some precision is lost so int is used if only whole numbers are needed.
String
Stores text values. Text values must be provided in "double quotes" to be a string. For example, "115" is a String, but 115 is an int).
boolean
Can only be true or false. Will be used extensively for control flow.

There are many more types in java. We will stick with these 4 for now.


    int items = 3;
    double price = 19.95;
    String salesPitch = "Buy one get two free! *additional charges may apply";
    boolean goodDeal = false;
== Printing
  • When you want to communicate with the human user
  • Can print any String to the console
  • Printing a variable will automatically convert a copy of its value to a String
Syntax: 

    System.out.print("Hello World!");

Will print Hello World! to the console.

Printing the value of a variable: 


    String hello;
    hello = "Hello World!";
    System.out.print(hello);

Same output as the previous example. However, the value being printed is retrieved from the variable named hello.

  • System.out.print
  • System.out.println

System.out.print prints its argument to the console exactly as its written. The following two examples produce exactly the same output:

Using one System.out.print statement: 


    System.out.print("Hello World!");

Using two System.out.print statements: 


    System.out.print("Hello Wo");
    System.out.print("rld!");

System.out.println prints its argument to the console just like System.out.print, but it also prints a newline character to the console. This means that the next thing to be printed will appear at the start of the next line.

Using two System.out.println statements: 


    System.out.print("Hello Wo");
    System.out.print("rld!");

produces two lines of output:

Hello Wo
rld!

== Expressions

An expressions is a portion of code that evaluates to a single value. 


    int x = 7;    // 7 is an expression, whose value is 7 (makes sense!)
    int y = 5;    // 5 is an expression, whose value is 5
    int z = x + y;  // x is an expression, whose value is whatever is stored in x (7 in this case)
                    // y is an expression, whose value is whatever is stored in y (5 in this case)
                    // x + y is an expression, whose value is the sum of the values of x and y
    System.out.println(z);  // z is an expression

Output: 12

In this example the expression x + y is an expression that is evaluated to a single value which is stored in the int variable named z. 


    int x = 9;
    int z;
    z = (x - 3) * 7;
    System.out.println(z);

Output: 42

Here (x - 3) * 7 is also an expression since it evaluates to a single value. Expressions can be use to combine any number of values into a single value. When using int or double we can perform arithmetic operations in the expressions, using operators like + (addition), - (subtraction), and * (multiplication).

The operator / denotes division, but division works a little differently for int values than it does for double values. For int values / computes the integer quotient. The corresponding remainder can be computed using the % operator. 


    int a = 15;
    int b = 6;
    System.out.println(a / b);
    System.out.println(a % b);
This will print the following: 
    2
    3
This is correct because 2 * 6 + 3 is 15.

For double values / computes the quotient (which may not be a whole number). For example, 


    double c = 15.0;
    double d = 6.0;
    System.out.println(c / d);
This will print: 
    2.5
This is correct because 2.0 * 6.0 is 15.0


To evaluate z = (x - 3) * 7; the following steps are executed by the computer.

  1. z = (x - 3) * 7; Replace all variable names with their stored values
  2. z = (9 - 3) * 7; Evaluate the sub-expression inside the parenthesis first
  3. z = 6 * 7; Finish evaluating the expression on the right side of the equal sign
  4. z = 42; Store the single resulting value in the variable named z
  5. 42; The assignment of z is also an expression as it returns the value being stored
  6. Execution is complete and the program moves to the next line of code
  • Expressions evaluate to a single value
  • Since expressions evaluate to a single value, they can be used wherever a value can be used

== String Concatenation

String Concatenation (combination) is an expression that combines multiple String values using the + operator. 


    System.out.println("hello " + "world");

Output: hello world 


    String example = "hello";
    String result;
    result = example + " " + "world";
    System.out.println(result);

Output: hello world

We combined 3 separate String values into a single String value and stored it in a variable called result. The value "hello" was recovered from the variable named example and concatenated with the values " " and "world" to create the String value "hello world" which is the value stored in result.

  1. result = example + " " + "world"; Replace all variable names with their stored values
  2. result = "hello" + " " + "world"; Evaluate each concatenation from left to right
  3. result = "hello " + "world";
  4. result = "hello world"; Store the single resulting value in the variable named result
  5. "hello world"; The assignment of result is also an expression as it returns the value being stored
  6. Execution is complete and the program moves to the next line of code
== Program Structure 

    public class LectureExample{

        public static void Q1(){
            String message = "I was called from main";
            System.out.println("Message from Q1: " + message);
        }

        public static void main(String[] args){
            Q1();
        }

    }

Output: Message from Q1: I was called from main

When the above program is ran execution starts with the code inside main. There is only one line of code in main which tells the computer to run the code in Q1 which print a message to the screen. Once the code is Q1 is executed, the computer returned to main to look for more code to run. It instead finds the end of main and program stops.
