--- short_title: names --- == Uses of names

Names are used for many things in a program, among them:

  • variables
  • methods
  • classes
  • packages
== Rules for identifiers

Technically a name is called an identifier. There are some hard and fast rules we must follow when creating identifiers. They are a little more intricate than we need to worry about for now (but you can read the gory details). For our purposes the following simplification will do:

  • Identifiers can contain UPPERCASE letters ('A' through 'Z'), lowercase letters ('a' through 'z'), digits ('0' through '9') and the underscore character ('_')
  • Identifiers cannot begin with a digit
  • Identifiers cannot be true, false, null, or any Java keyword.
See: == Conventions for identifiers

We use conventions to improve the readability of code.

Because identifiers cannot contain spaces multiword indentifiers can be difficult to read. Two common strategies to help programmers read and segment identifier names are:

  • separate distinct words in identifers by underscore characters
    • this_is_a_long_name
THIS_IS_A_DIFFERENT_LONG_NAME
  • use camelCase
    • thisIsAnotherLongName
ThisIsOurLastExampleOfALongName
== Conventions for variables

We use different conventions for different types of variables:

  • local variables (includes parameters)
    • camelCase
  • instance variables
    • camelCase or _camelCase
  • static variables
    • UPPER_CASE
