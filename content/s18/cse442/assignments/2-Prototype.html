== Practical Objective

Objective: Develop a running prototype that, by itself (no documentation or video required), clearly and simply conveys the idea of the final product to a non-technical audience. The prototype must also be easy to run for a technical user following instructions provided in the readme for the project (The readme can link to other resources)

Criteria:

  1. As a technical user, it was easy to run the software by following the readme instructions
  2. As a non-technical user, the running software alone gave me a clear understanding of the purpose of this software
== Prototype

Develop a running prototype of your project that shows the idea of the final product. The idea of the project must be shown through the software alone with no supporting documentation. For example, you cannot assume that the users have read your readme, wiki, or watched your video.

Once your prototype is complete, create a release from the develop branch of your prototype on GitHub as "v0.1.0"

No functionality is required at this point and most of the focus will be on the visuals and user interface. For example, you might have buttons that do not perform any actions, but it should be clear what the button will do after the software is developed and released. This prototype will become your final product by the end of the semester after your team implements all the functionality.

== YouTube Video

Upload a video demo of your prototype to YouTube that is shorter than 1 minute. This video must give an accurate demo of the current state of your project and show the running prototype. This video will be directed towards a non-technical audience to give people an idea of what you are going to accomplish this semester. You should not show any code or use any technical language in this video.

This video must be shorter than 1 minute. The goal is to quickly communicate your idea to an end user who may not be to run your software themselves.
