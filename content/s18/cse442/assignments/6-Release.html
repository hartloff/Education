== Practical Objective

Objective: Deploy intuitive and bug-free software with a professional landing page.

Verify: The landing page and software are designed for a non-technical audience. A non-technical user can be given a link to the landing page with no additional information and successfully use your software for its intended purpose. This implies that the landing page and software are intuitive to navigate and use. If any bugs are found (excluding extremely trivial or obscure bugs) then this objective has not been met.
Note: This objective is similar to the the beta release practical objective except your add-on feature must also meet these standards and only very minor bugs are allowed. Likewise, feedback from student users may be used as evidence for grading decisions, but will not directly determine any grades.

== Official Release
Technical Requirement: Deploy your software for non-technical users.
Verify: A non-technical user must be able to access and use your software for its intended purpose. This will replace your beta deployment.
Technical Requirement: Create a release from the master branch of your software on GitHub as "v1.0.0"
Verify: This release will appear on your project page.
Technical Requirement: Landing page and all documentation is updated to reflect the feature set in the official release.
Verify: Course staff will verify that the updates have ben made.

Complete your project, release it as "v1.0.0" on GitHub from the master branch, and maintain a live deployment of your software. You will release your software with the add-on feature completed in the previous phase which will replace your beta release. This will be the official release of your software and the last release for this course. For the remainder of the semester you must maintain this release and support your users.

For the official release, all documentation must be current and clear. Revise the readme, wiki, and landing page as needed.

Note that if your team has been completing all project tasks throughout the semester this might be a relaxing 2 weeks.

== Release Video
Requirement: Upload a video demo of your official release to YouTube.
Verify: This video will appear on your project page.

One last video. No time or content requirements this time. You decide how you want to present your final product to the world. This is the last video submission so this is the one you will want to show to potential employers. Your future self will thank the team if this video is great.
