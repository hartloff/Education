All those conditionals in 1 day was nuts!] == Code Blocks and Scope

Any time a section of code is surrounded by {braces} we call this section a code block. We saw code blocks corresponding to methods and For-Each loops and we will see several more uses of code blocks in this section. Code blocks are most commonly used in control flow to tell the program which sections of code should be executed together.

Variable Scope: Variables created inside a block of code are destroyed at the end of that block. 


    int number = 5;
    {
        double secondNumber = 7.43; // create a double variable and name it secondNumber
    } // secondNumber is destroyed here since it was created inside the block that is ending
    System.out.println(secondNumber); // This will cause an error since secondNumber has been destroyed

This code will cause a compiler error (the computer cannot understand how to run the code).

== Loops

What if we want to repeat a block of code multiple times?

Loops Concept Slides

While loop

Repeat a block of code while a condition is true. Once the condition is false, stop repeating the code.

Example: We want to set a thermostat to a desired temperature, but we can only increase the temperature 1 degree at a time. 


    int temperature=65;
    while(temperature<70){
        temperature = temperature+1;
        System.out.println("Temperature set to: " + temperature);
    }
    System.out.println("Final Temperature: " + temperature);

Output:

Temperature set to: 66
Temperature set to: 67
Temperature set to: 68
Temperature set to: 69
Temperature set to: 70
Final Temperature: 70

This could be done without a loop: 


    int temperature=65;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    System.out.println("Final Temperature: " + temperature);

This is repetitive, but it isn't too bad. However, this code is much less flexible than our code that uses a while loop. Let's assume we want to set the temperature to 75 instead of 70. With a while loop our example becomes: 


    int temperature=65;
    while(temperature<75){
        System.out.println("Temperature set to: " + temperature);
        temperature = temperature+1;
    }
    System.out.println("Final Temperature: " + temperature);

Output:

Temperature set to: 66
Temperature set to: 67
Temperature set to: 68
Temperature set to: 69
Temperature set to: 70
Temperature set to: 71
Temperature set to: 72
Temperature set to: 73
Temperature set to: 74
Temperature set to: 75
Final Temperature: 75

We barely had to change any code to have a large impact on the output. If we want the same output in the example without a loop we will have to change much more code: 


    int temperature=65;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    System.out.println("Final Temperature: " + temperature);

This code is less flexible and less readable than the code using a loop. What's worse is that we can't even write this code if we don't know how many degrees the temperature needs to be increased by. What if we want to write code that sets the temperature to 70 degrees regardless of the current temperature setting? How many times do we write "temperature = temperature+1;"?

  • Condition is checked before each iteration. If the condition is false when the loop is first reached it will never execute.
  • Condition should be false eventually! If the condition is never false the loop with run forever!

    int i=1;
    while(i<3){
        i++;
    }
    System.out.println("The value stored in i is " + i);

Output: The value stored in i is 3



Breakdown:

  1. int i=1; create an int variable, name it i, store the value 1
  2. while(i<3) Evaluate the boolean expression
  3. while(1<3)
  4. while(true) The condition is true so we will run the following block of code, then check this condition again
  5. {i++;} Run the block of code which increments i (add 1 to the value of i)
  6. while(i<3) After the end of the block, recheck the condition
  7. while(2<3)
  8. while(true) The condition is true so we will run the block of code again, then check this condition again
  9. {i++;} Run the block of code which increments i
  10. while(i<3) After the end of the block, recheck the condition
  11. while(3<3)
  12. while(false) The condition is false so the loop ends and control skips over the following block of code
  13. System.out.println("The value stored in i is " + i); Replace the variable name i with its value
  14. System.out.println("The value stored in i is " + 3); Convert the int value 3 to a String value
  15. System.out.println("The value stored in i is " + "3"); Concatenate the 2 String values
  16. System.out.println("The value stored in i is 3"); Display the resulting String value to the screen

For loop

Has the same power as while loops with different syntax. 


    for([initialization]; [condition]; [increment]){
        // body
    }

Where a while loop only needs a condition, for loops need 3 separate expressions, initialization code to be executed before the loop starts, the condition to be checked before each iteration, and an increment statement executed after each iteration.

  • Initialization: A statement that is executed before the loop begins. Initialization is only executed once.
  • Condition: The boolean expression that controls the number of iterations. The loop will run until the condition is false.
  • increment: Code to be executed each time the end of the loop is reached. This most commonly alters a value that is used in the condition to control the number of times the loop is executed. 

    for(int i=0; i<3; i++){
        System.out.println("The value stored in i is " + i);
    }

Output:

The value stored in i is 0
The value stored in i is 1
The value stored in i is 2

In this example we are initializing an int variable names i and setting it to 0 before the loop starts, Checking the condition "i<3" to control the loop, and increasing i by 1 after each iteration. This code is equivalent to the following. 


    {
        int i=0;
        System.out.println("The value stored in i is " + i);
        i++;
        System.out.println("The value stored in i is " + i);
        i++;
        System.out.println("The value stored in i is " + i);
        i++;
    }

Note that since the variable i is declared in the loop it will fall out of scope (be destroyed) when the loop is finished. 


    for(int i=3; i<5; i++){

== Iteration

We already have the tools we need to iterate over data structures by using the For-Each loop. Now we will see another way to iterate over an ArrayList by combining the size method, the get method, and a for loop. 


    static String charsToString(ArrayList<Character> inputList){
        // Concatenate all the Characters in the list into a single String.
        String result = "";
        for(int i=0; i<inputList.size(); i++){
            result = result + inputList.get(i);
        }
        return result;
    }

In this method we are given an ArrayList of Characters, but we have no information about the list. To learn about the the ArrayList we will call its methods and use the return values in our code. First, we know we'll need a loop that run once for each element of the ArrayList so we will ask inputList to tell us its size and use this value to setup our loop. We carefully setup this loop so the index variable i will exactly match the index values of the ArrayList. This way we can call inputList.get(i) in the loop and know it will be called for every element of the list.

We can create an ArrayList and call this method as follows: 


    ArrayList<Character> letters = new ArrayList<Character>();
    letters.add('l');
    letters.add('u');
    letters.add('c');
    letters.add('k');
    letters.add('y');
    String returnedString = charsToString(letters);
    System.out.println(returnedString);

Output: lucky

When charToString(letters) is called in this example it will run the code in the method with letters as inputList. Then the method code is executed. 


    String result = "";
    for(int i=0; i<inputList.size(); i++){
        result = result + inputList.get(i);
    }
    return result;

Since the inputList for this call contains 5 elements, its size method will return 5 and the code will become: 


    String result = "";
    for(int i=0; i<5; i++){
        result = result + inputList.get(i);
    }
    return result;

Simplifying this loop will yield: 


    String result = "";
    result = result + inputList.get(0);
    result = result + inputList.get(1);
    result = result + inputList.get(2);
    result = result + inputList.get(3);
    result = result + inputList.get(4);
    return result;

Then accessing the elements of the list will result in the final code that will be executed. 


    String result = "";
    result = result + 'l';
    result = result + 'u';
    result = result + 'c';
    result = result + 'k';
    result = result + 'y';
    return result;

Showing 1 step further we can see how the String result is being constructed 1 character at a time. 


    String result = "";
    result = "" + 'l';
    result = "l" + 'u';
    result = "lu" + 'c';
    result = "luc" + 'k';
    result = "luck" + 'y';
    return "lucky";

Now let's revisit Strings and explain something we've seen earlier in the course. 


    static String reverse(String input){
        // return the reverse of the string

        String reversed = "";
        for(int i=input.length()-1; i>=0; i--){
            reversed = reversed + input.charAt(i);
        }
        return reversed;
    }

Here we see how we can reverse a String using the tools we've seen earlier in the course. We are reversing the input String by iterating over its chars in reverse order while building a new String. This is very similar to the previous example except we don't use ArrayList and we iterate from the length of the String minus 1 to 0 instead from 0 to the length of the String minus 1.

== Boolean Expressions

The boolean value in the if statement can be a boolean expression that evaluates to a single boolean value.

Common relational operators:

  • Greater than: >
  • Less than: <
  • Equal: ==
  • Greater than or equal: >=
  • Less than or equal: <=
  • Not equal: !=


    boolean comparison = 5 > 10;
    System.out.println(comparison);

Output: false

boolean operators

boolean operators:

and: &&
Returns true only if both sides are true
or: ||
Returns true if either side is true
not: !
Returns the inverse of the following boolean value

    System.out.println(true && true); // true
    System.out.println(true && false); // false
    System.out.println(true || false); // true
    System.out.println(false || false); // false
    System.out.println(!true); // false

== If/else If/Else Concept Slides

One-armed conditional:

Check a boolean value. If true, run optional code. If false, skip the optional code.

Syntax: 

    boolean condition = true;
    if(condition){
        System.out.println("This line will be executed");
    }

Output: This line will be executed

Since the boolean value in the if statement is true, the code in the {}'s will be executed. 


    boolean condition = false;
    if(condition){
        System.out.println("This line will not be executed");
    }
    System.out.println("Control skips to here when the condition is false");

Output: Control skips to here when the condition is false

Since the boolean value in the if statement is false, the code in the {}'s will not be executed. 


    int number = 8;
    if(5 < number && number < 10){
        System.out.println("number is between 5 and 10");
    }

Output: number is between 5 and 10



Breakdown:

  1. if(5 < number && number < 10) Replace all variable names with their stored values
  2. if(5 < 8 && 8 < 10) Evaluate comparisons before boolean operators from left to right
  3. if(true && 8 < 10)
  4. if(true && true) Evaluate the boolean operator (and)
  5. if(true) The boolean expression results in true making the conditional for the if statement true
  6. {...} Execute the following code block

Else

Code to be executed on a false condition. 



    double priceOfDinner = 40.0;
    double currentCash = 20.0;
    System.out.println("Getting hungry");
    if(currentCash >= priceOfDinner){
        System.out.println("Eating a nice dinner!");
    } else {
        System.out.println("Eating Ramen");
    }
    System.out.println("Sleeping");

Output:

Getting hungry
Eating Ramen
Sleeping

Else if

Combination of else and if. Check multiple conditions. 


    double priceOfDinner = 40.0;
    double priceOfFastFood = 10.0;
    double currentCash = 20.0;
    System.out.println("Getting hungry");
    if(currentCash >= priceOfDinner){
        System.out.println("Eating a nice dinner!");
    } else if(currentCash >= priceOfFastFood){
        System.out.println("Eating Fast food");
    } else {
        System.out.println("Eating Ramen");
    }
    System.out.println("Sleeping");

Output:

Getting hungry
Eating Fast food
Sleeping

At most 1 block is executed in an if/else section.


    int number = 45;
    if(number < 30){
        System.out.println("number is less than 30");
    } else if(number < 40){
        System.out.println("number is less than 40");
    } else if(number < 50){
        System.out.println("number is less than 50");
    } else if(number < 60){
        System.out.println("number is less than 60");
    } else {
        System.out.println("number is greater than, or equal to, 60");
    }

Output:number is less than 50


The last condition number < 60 is true, but is never checked since a previous condition was true. Once a true condition is found the corresponding block of code is executed and control moves to the end of the entire group of conditionals.

Continue and Break

When you want to make an exception to the loops condition.

  • Continue: Skips the rest of the current iteration and moves to the increment statement
  • Break: Exit the loop entirely even if the condition is still met

    for(int i=0; i<7; i++){
        if(i == 4){
            continue;
        }
        System.out.println("The value stored in i is " + i);
    }

Output:

The value stored in i is 0
The value stored in i is 1
The value stored in i is 2
The value stored in i is 3
The value stored in i is 5
The value stored in i is 6

When i is 4 the continue is executed which skips to the end of the block without printing to the screen, then the continues as usual. 


    for(int i=0; i<7; i++){
        if(i == 4){
            break;
        }
        System.out.println("The value stored in i is " + i);
    }

Output:

The value stored in i is 0
The value stored in i is 1
The value stored in i is 2
The value stored in i is 3

When i is 4 the break is executed which immediately exits the loop even though i<7 is still true.
