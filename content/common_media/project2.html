The first part of the project was designed as a tutorial that walked you through the process of obtaining data from the Internet, processing the data, then visualizing that data on a web site that you can share with billions people. For part 2 you will be taking this same project structure and build an app of your own design. Requirements

You have a significant amount of flexibility to design this project however you'd like as long as you meet the following criteria.


The Data

Choose a data source that you access through a web API

You will choose a data source that will form the base of your app from a web API that we have not already used in this course (ie. 0 credit if you use the parking ticket API. Be creative). This data must be live and obtained with python code via an HTTP request made to the API each time a user visits your site.

Here are some lists of APIs for you to explore. You may use any API hosted on the Internet, though these lists should give you a good place to start

  • Buffalo Open Data
    • The city of Buffalo has recently released much of their data and host free APIs for developers to build apps. We used the parking ticket API from this project and their are many more APIs with a wealth of city data that is freely available
  • NYC Open Data
    • New York City also has an open data project with many freely available APIs
  • US Open Data
    • Open data at the federal level for the United States. If you want to analyze national trends you will find many interesting data sets here
  • Other Government Data
    • Not interested in Buffalo, NYC, or US data? Check if your home city/country has an open data program. These programs have increased in popularity over the past few years and contain data that could improve policies, inform citizens, and even save lives if analyzed properly
  • Big List of Web APIs
    • With over 500 APIs you will find something that interests you in this list. Note that some of these APIs are not hosting data but instead allow you to interact with their app (ex. Google Calendar API). For this project you must analyze data which will exclude some of the APIs on this list
  • Any other web API
    • If you find an API that is not included anywhere in these lists, feel free to use it

Process The Data

On your server (in python), process the data to prepare it for visualization

For example, in part 1 we removed any parking tickets that did not contain a location and removed any fields that were not required for visualization. Recall that the API returns a lot of information about each ticket, but we only send latitude, longitude, and description to the front-end of the app.

This processing does not have to be complex to meet this criteria, but you cannot send the results from the API unmodified to your front-end. This may be a simple filter, but your Python code has to process the data from the API before sending it to the front-end for visualization.


Visualize The Data

On the front-end of your web app, visualize the data with at least 2 different visualizations

In part 1 we created one visual which was a map with the parking tickets placed at their locations with their descriptions. In Infinite Questions you built web pages that displayed line/scatter and pie charts. For this part of the project you will choose 2 different (you can't make 2 of the same type of visual. Two maps == 0 credit) visuals to display the processed data that you obtain from your Python server. Note that to make this interesting you may want to have Python process the data in 2 different ways.

Each visual must be generated by a JavaScript library. For example, adding an image to your page using an <img> element or displaying raw text from the API will not count.

Recall that Plotly offers a large number of visuals to choose from.

Additional Criteria

In addition to the requirements above you must have one additional aspect to your project. You may choose on aspect from the following list to add. Note that you only need one of these, not all of them, though we won't stop you from implementing more than one.

  • Add communication between users with persistent storage (File or database) on your server (Ex. A comments section)
  • Use a web framework other than bottle (Ex. Flask, Django, Tornado). If you choose this you are expected to learn one of these frameworks by studying the documentation as with other aspects of the course
  • Make a game out of the data using a game framework like phaser
  • Use 2 different data sets instead of just one and combine them on your front-end. At least one of your visuals must contain information from both data sets (do not just create one plot for each data set). The data sets must come from 2 different APIs (ie. plotting tweets containing #x and tweets containing #y on the same graph will not count).
  • Add a secure login system to your app and have at least 1 feature that utilizes this login
  • Scrape data from an HTML web site instead of using an API. You must still connect to the live site from your server using Python code, but the site will host HTML and you will scrape the live data using Python to extract the data you want to process
  • Other. If you have an idea that is not on this list you can message the instructors on Piazza and we will consider it for approval. If approved it will be added to this list.
